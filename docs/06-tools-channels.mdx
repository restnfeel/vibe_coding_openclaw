# 도구 & 채널 연동

> OpenClaw의 도구(Tools) 시스템과 30+ 메시징 채널 연동을 다룹니다.

---

## Tools — 에이전트의 도구 상자

### 도구 프로필

도구 접근 범위를 프로필로 관리합니다.

| 프로필 | 포함 범위 | 사용 상황 |
|--------|----------|----------|
| `minimal` | 세션 상태 조회만 | 최소 권한 필요 시 |
| `coding` | 파일, 런타임, 세션, 메모리, 이미지 | 개발 작업 |
| `messaging` | 메시징 채널, 세션 관리 | 채팅봇 운영 |
| `full` | 모든 도구 | 제한 없는 전체 접근 |

```json5
// openclaw.json
{
  tools: {
    profile: "coding",

    // 추가 허용
    alsoAllow: ["browser", "web_search"],

    // 명시적 차단 (deny가 allow보다 우선)
    deny: ["exec"]
  }
}
```

### 핵심 도구 카테고리

#### 실행 & 프로세스

| 도구 | 용도 |
|------|------|
| `exec` | 셸 명령 실행 (백그라운드, 타임아웃, 권한 상승 지원) |
| `process` | 백그라운드 세션 관리 (목록, 로그, 종료 등) |

#### 파일 작업

| 도구 | 용도 |
|------|------|
| `read` | 파일 읽기 |
| `write` | 파일 쓰기 |
| `edit` | 파일 편집 |
| `apply_patch` | 여러 파일에 구조화된 패치 적용 |

#### 웹 도구

| 도구 | 용도 |
|------|------|
| `web_search` | Brave Search API 웹 검색 |
| `web_fetch` | URL 콘텐츠 가져오기 (HTML → 마크다운 변환) |

#### 브라우저 자동화

| 도구 | 용도 |
|------|------|
| `browser` | 전용 브라우저 (탭 관리, 네비게이션, 스냅샷) |
| `act` | UI 상호작용 (클릭, 타이핑, 호버, 드래그 등) |

#### 세션 & 서브에이전트

| 도구 | 용도 |
|------|------|
| `sessions_list` | 활성 세션 목록 |
| `sessions_history` | 세션 기록 조회 |
| `sessions_send` | 다른 세션에 메시지 전송 |
| `sessions_spawn` | **서브에이전트 생성** (파일 첨부, 스레드 바인딩 가능) |

#### 스케줄링

| 도구 | 용도 |
|------|------|
| `cron` | 예약 작업 관리 (일회성, 반복, cron 표현식) |

### 도구 그룹 단축키

설정에서 그룹 이름으로 여러 도구를 한 번에 제어할 수 있습니다.

```json5
{
  tools: {
    deny: ["group:runtime"],   // exec, bash, process 일괄 차단
    allow: ["group:fs"]        // read, write, edit, apply_patch 일괄 허용
  }
}
```

### 프로바이더별 도구 정책

특정 AI 프로바이더/모델에 대해 별도 도구 정책을 설정할 수 있습니다.

```json5
{
  tools: {
    byProvider: {
      "openai/*": {
        deny: ["group:runtime"]  // OpenAI 모델에서는 실행 도구 차단
      }
    }
  }
}
```

## 채널 연동

### 지원 채널 목록

OpenClaw는 30개 이상의 메시징 플랫폼을 지원합니다.

| 카테고리 | 채널 |
|---------|------|
| 인기 | WhatsApp, Telegram, Discord, Slack |
| 기업용 | Microsoft Teams, Mattermost |
| 보안 | Signal, iMessage |
| 오픈소스 | Matrix, IRC |
| 기타 | Nostr 등 |

### WhatsApp 설정

```json5
// openclaw.json
{
  channels: {
    whatsapp: {
      dmPolicy: "pairing",           // 페어링 코드 기반 접근
      allowFrom: ["+821012345678"],   // 허용 전화번호
      groupPolicy: "allowlist",       // 그룹 접근 정책
      groupAllowFrom: ["+821012345678"]
    }
  }
}
```

```bash
# WhatsApp 연결
openclaw channels login --channel whatsapp
```

QR 코드를 스캔하여 연결합니다.

### Telegram 설정

1. [@BotFather](https://t.me/BotFather)에서 봇 생성
2. 토큰 설정:

```bash
openclaw config set channels.telegram.token "YOUR_BOT_TOKEN"
```

### Discord 설정

1. [Discord Developer Portal](https://discord.com/developers/applications)에서 봇 생성
2. **Message Content Intent** 활성화
3. 토큰 설정:

```bash
# 환경 변수 방식
export DISCORD_BOT_TOKEN="YOUR_TOKEN"

# 또는 설정 파일 방식
openclaw config set channels.discord.token "YOUR_TOKEN"
```

### DM 접근 모델

| 모델 | 설명 | 보안 수준 |
|------|------|----------|
| `pairing` (기본) | 시간 제한 페어링 코드 | 높음 |
| `allowlist` | 허용 목록 기반 | 높음 |
| `open` | 누구나 접근 가능 | 낮음 |
| `disabled` | DM 비활성화 | 최고 |

## 보안 설정

### 강화된 보안 설정 예시

```json5
{
  tools: {
    profile: "messaging",
    deny: ["group:automation", "group:runtime", "group:fs", "sessions_spawn"],
    fs: { workspaceOnly: true },
    exec: { security: "deny", ask: "always" }
  }
}
```

### Docker 샌드박싱

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "all",          // 모든 도구 샌드박싱
        scope: "agent",       // 에이전트 단위 격리
        workspaceAccess: "none"
      }
    }
  }
}
```

### 보안 감사

```bash
# 보안 감사 실행
openclaw security audit --deep

# 자동 수정 포함
openclaw security audit --deep --fix
```
