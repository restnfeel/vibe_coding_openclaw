# Skills 만들기 — 실습 가이드

> 이 페이지가 가이드의 **핵심**입니다.
> 직접 Skills를 만들어 보면서 OpenClaw의 가장 강력한 기능을 체험합니다.

---

## 실습 1: Hello World 스킬

가장 간단한 스킬을 만들어 봅니다.

### Step 1: 디렉토리 생성

```bash
mkdir -p ~/.openclaw/workspace/skills/hello-world
```

### Step 2: SKILL.md 작성

```bash
cat > ~/.openclaw/workspace/skills/hello-world/SKILL.md << 'EOF'
---
name: hello_world
description: A simple skill that says hello.
---

# Hello World Skill

When the user asks for a greeting, use the `echo` tool to say
"Hello from your custom skill!".
EOF
```

### Step 3: 스킬 새로고침

에이전트에게 "refresh skills"라고 말하거나, Gateway를 재시작합니다.

```bash
openclaw gateway restart
```

### Step 4: 테스트

```bash
openclaw agent --message "use my new skill"
```

에이전트가 "Hello from your custom skill!" 이라고 응답하면 성공입니다!

---

## SKILL.md 작성법 상세

### 기본 구조

```markdown
---
name: skill_identifier
description: 이 스킬이 하는 일을 한 줄로 설명
---

# 스킬 제목

여기에 에이전트가 따라야 할 지시를 마크다운으로 작성합니다.
도구 사용법, 출력 형식, 제약 조건 등을 명시합니다.
```

### 프론트매터 규칙

- YAML 프론트매터는 **단일 행 키-값 쌍**만 지원합니다
- `metadata`는 한 줄 JSON 객체여야 합니다
- `{baseDir}`을 사용하면 스킬 폴더 경로를 참조할 수 있습니다

## 실습 2: 코드 리뷰 스킬

실무에서 바로 쓸 수 있는 코드 리뷰 스킬을 만들어 봅니다.

### SKILL.md

```markdown
---
name: code_review
description: 코드 변경사항을 리뷰하고 개선점을 제안합니다.
---

# 코드 리뷰 스킬

## 사용 시점
사용자가 코드 리뷰를 요청하면 이 스킬을 적용합니다.

## 리뷰 절차

1. `exec` 도구로 `git diff` 실행하여 변경사항 확인
2. 변경된 파일을 `read` 도구로 읽기
3. 아래 기준에 따라 리뷰 작성

## 리뷰 기준

| 카테고리 | 확인 사항 |
|---------|----------|
| 보안 | SQL 인젝션, XSS, 하드코딩된 시크릿 |
| 성능 | N+1 쿼리, 불필요한 루프, 메모리 누수 |
| 가독성 | 네이밍, 함수 크기, 주석 필요성 |
| 테스트 | 테스트 커버리지, 엣지 케이스 |

## 출력 형식

```text
## 코드 리뷰 결과

### 요약
- 전체 평가: [좋음/보통/개선필요]
- 변경 파일 수: N개

### 발견사항
1. [심각도] 파일명:라인 — 설명
2. ...

### 개선 제안
- ...
```
```

### 테스트

```bash
# Git 저장소에서 테스트
openclaw agent --message "최근 커밋을 코드 리뷰해줘"
```

## 실습 3: API 엔드포인트 생성 스킬

반복적인 CRUD API 생성을 자동화하는 스킬입니다.

### SKILL.md

```markdown
---
name: fastapi_crud
description: FastAPI CRUD 엔드포인트를 자동 생성합니다.
---

# FastAPI CRUD 생성 스킬

## 사용 시점
사용자가 새 API 엔드포인트나 CRUD 기능을 요청하면 적용합니다.

## 기술 스택
- FastAPI + SQLAlchemy 2.0 (async)
- Pydantic v2 스키마
- PostgreSQL
- pytest + httpx 테스트

## 생성 파일 구조

```text
app/{domain}/
├── router.py     — FastAPI 라우터
├── service.py    — 비즈니스 로직
├── schema.py     — Pydantic 모델
├── model.py      — SQLAlchemy 모델
└── test_{domain}.py — 테스트
```

## 코딩 규칙
- 모든 함수에 타입 힌트 적용
- 에러 핸들링: HTTPException + custom ErrorResponse
- 페이지네이션: offset/limit 패턴
- 소프트 삭제 패턴 적용 (deleted_at 컬럼)

## 출력 순서
1. model.py → 2. schema.py → 3. service.py → 4. router.py → 5. test 파일
```

## 실습 4: 슬래시 커맨드 스킬

`user-invocable` 설정으로 슬래시 커맨드(`/명령어`)로 직접 호출 가능한 스킬을 만듭니다.

### SKILL.md

```markdown
---
name: deploy_check
description: 배포 전 체크리스트를 실행합니다.
user-invocable: true
---

# 배포 전 체크 스킬

사용자가 `/deploy_check`을 입력하면 아래 단계를 자동 실행합니다.

## 실행 순서

1. `exec` 도구로 `npm test` 실행 → 테스트 통과 확인
2. `exec` 도구로 `npm run build` 실행 → 빌드 성공 확인
3. `exec` 도구로 `npm run lint` 실행 → 린트 에러 확인
4. 결과 요약을 아래 형식으로 출력

## 출력 형식

| 항목 | 상태 | 비고 |
|------|------|------|
| 테스트 | PASS/FAIL | 실패 시 원인 |
| 빌드 | PASS/FAIL | 실패 시 원인 |
| 린트 | PASS/FAIL | 경고 수 |

모든 항목이 PASS이면 "배포 준비 완료!" 메시지를 출력합니다.
```

## 실습 5: 도구 직접 실행 (command-dispatch)

모델을 거치지 않고 도구를 직접 실행하는 스킬입니다.

```markdown
---
name: quick_status
description: 시스템 상태를 빠르게 확인합니다.
user-invocable: true
command-dispatch: tool
command-tool: exec
command-arg-mode: raw
---

# Quick Status

`/quick_status`를 입력하면 모델 호출 없이 바로 `exec` 도구가 실행됩니다.

기본 실행 명령: `openclaw gateway status --deep --json`
```

이 방식은 모델 호출 비용 없이 도구를 직접 실행하므로, 자주 쓰는 명령에 적합합니다.

## 실습 6: 보조 파일 활용

Skills 디렉토리에 스크립트나 템플릿을 함께 넣고 `{baseDir}`로 참조할 수 있습니다.

### 디렉토리 구조

```
skills/report-generator/
├── SKILL.md
├── template.html       ← 보고서 템플릿
└── generate.sh         ← 보조 스크립트
```

### SKILL.md

```markdown
---
name: report_generator
description: 프로젝트 진행 보고서를 생성합니다.
---

# 보고서 생성 스킬

## 참고 파일
- 보고서 템플릿: `{baseDir}/template.html`
- 생성 스크립트: `{baseDir}/generate.sh`

## 실행 절차
1. `read` 도구로 `{baseDir}/template.html` 읽기
2. 프로젝트 현재 상태 수집 (git log, 이슈 목록 등)
3. 템플릿에 데이터 채워서 보고서 생성
4. `write` 도구로 `reports/` 디렉토리에 저장
```

## Skills 작성 모범 사례

### DO (권장)

| 항목 | 설명 |
|------|------|
| 명확한 사용 시점 | "사용자가 X를 요청하면" 형태로 명시 |
| 구체적인 출력 형식 | 테이블, 코드 블록 등 형식을 정의 |
| 제약 조건 명시 | 사용 기술 스택, 금지 패턴 등 |
| 단계별 절차 | 번호 매긴 순서대로 실행 지시 |
| 도구 이름 명시 | `exec`, `read`, `write` 등 구체적 도구명 |

### DON'T (주의)

| 항목 | 이유 |
|------|------|
| 너무 긴 지시문 | 토큰 낭비, 핵심이 묻힘 |
| 모호한 표현 | "적절히 처리" → 구체적으로 변경 |
| 여러 역할 혼합 | 한 스킬에 한 역할만 |
| 하드코딩된 경로 | `{baseDir}` 사용 권장 |

## 스킬 디버깅 팁

### 1. 로드 여부 확인

에이전트에게 "현재 로드된 스킬 목록을 보여줘"라고 요청합니다.

### 2. 새로고침

```bash
# 방법 1: 에이전트에게 요청
"refresh skills"

# 방법 2: Gateway 재시작
openclaw gateway restart
```

### 3. 실시간 리로드 활성화

```json5
{
  skills: {
    load: {
      watch: true,
      watchDebounceMs: 250
    }
  }
}
```

이 설정을 켜면 SKILL.md 파일 수정 시 자동으로 리로드됩니다.
