# CloudType으로 OpenClaw 배포하기

> [CloudType](https://cloudtype.io/ko)은 GitHub 연동만으로 배포가 완료되는 한국형 클라우드 플랫폼입니다.
> 보안이 확보된 환경에서 OpenClaw를 운영할 수 있는 실습 환경으로 활용합니다.

---

## CloudType을 선택한 이유

| 장점 | 설명 |
|------|------|
| 간편한 배포 | GitHub 저장소 연결 → 자동 빌드 → 자동 배포 |
| 보안 환경 | HTTPS 자동 발급, 격리된 컨테이너 실행 |
| 무료 티어 | 1GB 메모리, 4개 서비스 동시 실행 가능 |
| 한국어 지원 | UI와 문서 모두 한국어 |
| 다양한 템플릿 | Node.js, Docker 등 100+ 템플릿 |

---

## 사전 준비

1. [CloudType 계정](https://cloudtype.io/ko) 생성
2. GitHub 계정 연동
3. OpenClaw 프로젝트가 GitHub에 푸시된 상태

---

## 배포 방법 A: Docker 기반 (권장)

### Step 1: Dockerfile 작성

프로젝트 루트에 `Dockerfile`을 생성합니다.

```dockerfile
FROM node:22-slim

WORKDIR /app

# OpenClaw 설치
RUN npm install -g openclaw@latest

# 설정 디렉토리 생성
RUN mkdir -p /root/.openclaw

# 설정 파일 복사 (있는 경우)
COPY openclaw.json /root/.openclaw/openclaw.json

# Skills 복사 (있는 경우)
COPY skills/ /root/.openclaw/workspace/skills/

# Gateway 포트
EXPOSE 18789

# 시작 명령
CMD ["openclaw", "gateway", "--port", "18789"]
```

### Step 2: openclaw.json 설정

```json5
{
  gateway: {
    port: 18789,
    auth: {
      token: "${OPENCLAW_GATEWAY_TOKEN}"
    }
  },
  tools: {
    profile: "messaging"
  },
  agents: {
    defaults: {
      model: {
        primary: "anthropic/claude-sonnet-4-20250514"
      }
    }
  }
}
```

### Step 3: CloudType에서 배포

1. CloudType 대시보드 → **새 프로젝트 생성**
2. **GitHub 저장소 연결** → OpenClaw 프로젝트 선택
3. **빌드 방식**: Docker 선택
4. **환경 변수** 설정:

| 변수 | 값 | 설명 |
|------|------|------|
| `OPENCLAW_GATEWAY_TOKEN` | (비밀 값) | Gateway 인증 토큰 |
| `ANTHROPIC_API_KEY` | (비밀 값) | AI 모델 API 키 |

5. **포트**: `18789` 설정
6. **배포** 클릭

## 배포 방법 B: Node.js 기반

Docker 없이 Node.js 템플릿으로 배포할 수도 있습니다.

### package.json 설정

```json
{
  "name": "openclaw-server",
  "scripts": {
    "start": "openclaw gateway --port ${PORT:-18789}",
    "postinstall": "npm install -g openclaw@latest"
  },
  "engines": {
    "node": ">=22"
  }
}
```

### CloudType 설정

1. 빌드 방식: **Node.js** 선택
2. 시작 명령: `npm start`
3. 환경 변수 동일하게 설정

## CloudType 무료 티어 제한사항

| 항목 | 제한 |
|------|------|
| 메모리 | 1 GB |
| 디스크 | 1 GB (임시) |
| 동시 서비스 | 4개 |
| 네트워크 | 5 Mbps |
| 월 트래픽 | 10 GB |
| 빌드 시간 | 200분/월 |
| **서비스 중단** | **하루 1회 자동 중단** |

:::warning 무료 티어 주의사항
무료 티어에서는 하루에 한 번 서비스가 자동으로 중단됩니다.
실습/테스트 용도로는 충분하지만, 프로덕션 운영에는 유료 플랜(Hobby: 월 6,600원~)을 권장합니다.
:::

## 배포 후 확인

### 1. 서비스 상태 확인

CloudType 대시보드에서:
- **로그 뷰어**: 실시간 로그 확인
- **터미널**: 컨테이너 접속 가능
- **메트릭스**: CPU/메모리 사용량

### 2. Gateway 접속

CloudType이 할당한 도메인으로 접속:

```
https://your-project.cloudtype.app/
```

### 3. 원격에서 에이전트 테스트

```bash
# Gateway 상태 확인
curl https://your-project.cloudtype.app/api/status \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## 보안 체크리스트

| 항목 | 확인 |
|------|------|
| Gateway 인증 토큰 설정 | `OPENCLAW_GATEWAY_TOKEN` 환경 변수 |
| API 키를 환경 변수로 관리 | 코드에 하드코딩 금지 |
| HTTPS 접근만 허용 | CloudType이 자동 처리 |
| 도구 프로필 제한 | `messaging` 또는 `minimal` 사용 |
| DM 정책 설정 | `pairing` 또는 `allowlist` |
| 실행 도구 차단 | 운영 환경에서는 `exec` 차단 권장 |
